Measures the agent, not just the model.

The harness runs real workflows that require search, fetching, code execution, support-data lookups, citation grounding, and trace normalization.

Task completion

Checks whether the final answer satisfies task-specific outcomes.

Tool accuracy

Scores required calls, argument quality, errors, and unnecessary thrash.

Citation quality

Verifies cited URLs came from tool traces and support stated claims.

Latency

Normalizes elapsed runtime against category-level budgets.

Cost

Uses a shared pricing path so totals are comparable run to run.

Failure rate

Tracks hard errors, timeouts, and max-tool-call stops.

Same tools. Same schemas. Same execution code.

Both providers receive the same benchmark tools. The adapters only translate a shared tool spec into the SDK-specific shape.

1

Load task YAML

Each task declares tools, expected outcomes, scoring weights, and limits.

2

Expose shared tools

OpenAI and Anthropic receive identical JSON Schema from one registry.

3

Normalize traces

Messages, tool calls, usage, latency, and errors become one trace format.

4

Write JSON results

Timestamped files can be shown, compared, replayed, or read by a dashboard.

A benchmark contract that survives the UI.

The first release is CLI-only. The result schema is the product surface: every run writes a portable scorecard with normalized traces and cassette data.

Install it like a normal CLI.

Published on PyPI for isolated installation with pipx. Add provider keys, run tasks, then compare result files in the terminal. 

pipx install agent-builder-evals
agent-evals list-tasks

export OPENAI_API_KEY="..."
export YOU_API_KEY="..."

agent-evals run \
  --provider openai \
  --tasks coding_001,company_research_001
agent-evals show results/run_openai.json

agent-evals compare \
  results/run_openai.json \
  results/run_anthropic.json

agent-evals replay results/run_openai.json
